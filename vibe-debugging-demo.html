üí° Use Cases

1. Post-Feature Quality Check

After implementing a feature, validate the commit to ensure all intended files were modified and no unexpected changes were made.

2. Session Review & Context Drift Monitoring

At the end of a coding session, validate recent commits to check for declining alignment scores that indicate growing context drift.

3. Bug Root Cause Analysis

When debugging, review validation reports of commits related to the bug to identify if misalignment introduced the issue.

4. Cross-LLM Debugging (Vibe Transfer)

Use validation data to query multiple LLMs with precise context about misalignments, getting diverse perspectives for faster resolution.

üìà Alignment Scoring

Alignment score (0.0 - 1.0) is calculated based on weighted factors:

File Overlap (Expected vs Actual) 40% weight
Prompt Quality (Length & Specificity) 30% weight
Agent Plan Presence 15% weight
Change Size Reasonableness 15% weight

0.7 - 0.9
